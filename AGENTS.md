# AGENTS.md

本文件用于约束 DecisionJury 项目中所有成员使用 AI 辅助开发的行为。所有 AI 生成内容都必须经过人工理解、运行和检查后才能合并。

## 语言

- 内部代码命名、提交信息和技术注释优先使用英文。
- 面向课程汇报、项目文档和团队沟通优先使用简体中文。

## 开发原则

- 只实现当前 Milestone 要求的功能，不提前扩展医疗、法律、金融、情感等高风险场景。
- 任何新增功能都要能映射到 MVP、SPEC 或 Issue。
- 不提交无法运行的代码、不提交未说明用途的大段 AI 生成代码。
- 不把 API Key、Token、Cookie、数据库文件、向量索引产物提交到仓库。

## AI 使用规矩

- 使用 AI 前先说明目标、输入、约束和期望输出。
- 使用 AI 生成代码后，必须自己阅读并确认逻辑。
- 使用 AI 修改代码后，必须运行相关测试或手动验证。
- 使用 AI 生成文档后，必须检查是否符合项目真实范围。
- 禁止让 AI 编造已经完成的功能、测试结果、部署地址或市场数据。

## Git 协作规矩

- `main` 分支只保留稳定版本。
- 每个成员从 `dev` 创建自己的功能分支，命名格式为 `feature/<module-name>`。
- 合并代码必须通过 Pull Request。
- PR 描述必须包含：改动内容、验证方式、影响范围、未完成事项。
- 不使用 `git reset --hard`、强推或删除他人分支，除非全组明确同意。

## Definition of Done

一个功能完成必须同时满足：

- 功能能在本地运行。
- 关键路径有测试或明确的手动验收记录。
- 文档或接口说明已同步更新。
- 不包含密钥、临时文件、无关产物。
- PR 已被至少一名组员检查。
